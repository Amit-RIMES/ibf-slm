⚡ What is a Trigger?

A trigger is a threshold rule that fires automatically when an observed or forecast variable crosses a pre-defined level. For example: "Fire if SPI-3 ≤ −1.5" or "Fire if forecast mean precipitation ≥ 200 mm."

When a trigger fires it creates a Trigger Activation — a time-stamped record of the event and the value that exceeded the threshold. The IBF Matrix shows all active triggers in a hazard × probability grid.

Tip: Go to Triggers to create or edit threshold rules. Each trigger can be linked to one or more alert channels so stakeholders are notified automatically when it fires.

Key terms:

  • Active — the trigger has fired and has not been acknowledged.
  • Acknowledged — a duty officer has reviewed and confirmed the event.
  • Impact verdict — the post-event assessment: Hit / Partial / False Alarm. Set this in Verification after the event passes.

📉 Reading the SPI Drought Index

The Standardised Precipitation Index (SPI) measures rainfall deficit or surplus relative to the long-term climatological average for the same month and location. It is dimensionless — a value of −1.0 means one standard deviation drier than average; +1.0 means one standard deviation wetter.

Timescales:

  • SPI-1 — single month. Responds quickly; useful for flash drought or wet spells. High month-to-month variability.
  • SPI-3 — 3-month accumulated. Captures seasonal drought onset.
  • SPI-6 — 6-month accumulated. Best indicator of medium-term agricultural drought. Smoother and more persistent.

Interpretation thresholds:

  • 0 to −0.99 — Mildly dry (no alert warranted)
  • −1.0 to −1.49 — Moderately dry
  • −1.5 to −1.99 — Severely dry
  • ≤ −2.0 — Extremely dry (corresponds to roughly the driest 2.5% of historical months)
Low confidence flag: When fewer than 5 reference years exist for a given month, the SPI card is labelled "low confidence." Use these values with caution and do not trigger alerts based on them alone.

The drift chart on the Drought page shows SPI over time. A sustained negative trend (slope moving left-to-right downward) indicates developing drought even before the value crosses −1.0.

🚦 WMO Warning Levels

The IBF app uses the WMO 4-level colour-coded warning framework. The overall risk level is computed automatically from the SPI score, seasonal outlook, and active trigger activations. It is displayed on the Dashboard, IBF Matrix, and Shift Handover page.

LevelColourMeaningTypical action
Green Low No significant hazard. Conditions within normal range. Routine monitoring.
Yellow Moderate Developing anomaly. Threshold(s) may be approached. Increase data collection frequency; brief stakeholders.
Orange High One or more triggers active. Significant impact likely. Issue advisory bulletin; activate early action protocols.
Red Extreme Multiple triggers active. Severe or catastrophic impact expected. Issue urgent bulletin; escalate to emergency management.
The scoring uses: SPI anomaly points (max 30) + seasonal outlook points (max 20) + active trigger activation points (max 50). Levels: Green ≤ 25, Yellow ≤ 50, Orange ≤ 75, Red > 75.

📄 Submitting a Bulletin for Approval

Bulletins are generated on-demand from live data and must go through a review workflow before distribution. The workflow has three steps:

  1. Generate — Go to Bulletin, choose a data source, reporting window (7–90 days), and optional title. Select the output language if your stakeholders use French, Spanish, Portuguese, or Bahasa Indonesia. Click Generate Bulletin.
  2. Submit for review — From Bulletin → Drafts, click Submit on a pending draft. The system emails all admins for approval.
  3. Approve and send — An admin reviews, optionally adds approval notes, then clicks Approve. Once approved, click Send to distribute to all active subscribers.
Tip: Admins can also use Bulletin → Schedule to configure automatic daily or weekly bulletin emails to a subscriber list — no manual generate/send step needed.

The Print / PDF button opens a print-optimised version in a new tab. Use your browser's print dialog to save as PDF for offline sharing.

📱 Setting Up SMS / WhatsApp Alerts

When a trigger fires, the system can send immediate SMS or WhatsApp messages to a list of recipients via Twilio. Each trigger can have its own recipient list and message template.

  1. Configure credentials (admin only) — Go to Admin → SMS and enter your Twilio Account SID, Auth Token, and sender number (SMS) or WhatsApp sender (e.g. whatsapp:+14155238886).
  2. Create an alert channel — Go to Alerts and click Add Channel. Choose type (SMS or WhatsApp), add recipient numbers, and write a message template. Use {trigger}, {value}, {threshold}, and {hazard} as placeholders.
  3. Link to a trigger — Edit the trigger ( Triggers) and select the alert channel in the Alert channel dropdown. The channel fires automatically whenever the trigger activates.
Note: WhatsApp Business API messages must be sent using approved templates when messaging users who have not messaged you in the last 24 hours. Session messages (replies within 24 h) can use free-form text.

✅ Forecast Verification

Forecast verification is the process of checking whether a trigger activation (forecast-based alert) actually corresponded to a real impact on the ground. Regular verification improves trigger thresholds over time and satisfies donor accountability requirements.

Go to Verification to see all past trigger activations.

Verdict options:

  • Hit — The alert was justified; the hazard caused the expected impact.
  • Partial — The hazard occurred but impact was less severe than expected, or affected a smaller area.
  • False Alarm — The threshold was crossed but no meaningful impact was observed.

Skill metrics shown at the top of the page:

  • Hit Rate — proportion of verified activations rated Hit or Partial. Higher is better.
  • False Alarm Rate — proportion rated False Alarm. Lower is better.
Tip: Use the filter bar to focus on a specific hazard type or date range. Your verdict and notes are saved immediately; you can edit them later if new information comes in.

🔄 Shift Handover

The Shift Handover page (Reports → Handover) provides a one-page situational summary for duty officer shift changes. It draws on live data — warning level, active triggers, recent activations, latest bulletin, SPI status, and recent impacts.

  1. Open the Handover page at the end of your shift. Review the current warning level and any unverified activations.
  2. Fill in the Outgoing officer and Incoming officer names, and the handover time (client-side only — not saved to the database).
  3. Add any situational notes in the Handover notes text area — e.g. data gaps, pending actions, or stakeholder communications in progress.
  4. Click Copy as text to copy a plain-text summary to the clipboard (for pasting into a shift log or messaging app), or click Print to produce a PDF record.
Important: The officer names and notes are not saved to the database — they exist only in your browser tab. If you close or refresh the page before copying or printing, they will be lost.

🛰 Data Sources

The IBF App ingests data from several external sources:

  • CHIRPS (Climate Hazards Group InfraRed Precipitation with Station data) — 5-day and monthly observed rainfall at 0.05° resolution. Updated via Observed or the Auto-Sync scheduler.
  • ECMWF Open Data — 15-day global ensemble precipitation forecasts. Fetched from the ECMWF open data S3 endpoint (no API key required for the free tier). Managed via ECMWF.
  • Copernicus CDS — ERA5 reanalysis and seasonal forecast data (SEAS5). Requires a CDS API key. Managed via CDS.
  • GloFAS — Global Flood Awareness System river discharge forecasts. Requires a Copernicus account. Managed via GloFAS.
  • Manual station data — Rainfall observations from weather stations can be uploaded as CSV or entered manually via Stations.
Auto-Sync (Auto-Sync) lets you configure a scheduled task that automatically fetches the latest CHIRPS data and recomputes SPI values on a set interval — without manual intervention.
